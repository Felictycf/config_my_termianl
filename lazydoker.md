

### 🐳 LazyDocker 核心快捷键指南

LazyDocker 的面板通常分为：Services, Containers, Images, Volumes。

#### 1. 🧭 导航 (与 LazyGit 一样)
* **`[` / `]`** 或 **`h` / `l`**: 左右切换面板。
* **`k` / `j`**: 上下移动列表。
* **`?`**: 查看帮助。

#### 2. 📦 容器操作 (Containers 面板)
这是最常用的面板。

| 快捷键 | 作用 | 说明 |
| :--- | :--- | :--- |
| **`Enter`** | **聚焦日志** | 在右侧最大化显示日志，便于查看报错 |
| **`s`** | **停止** (Stop) | 停止当前选中的容器 |
| **`r`** | **重启** (Restart) | 重启容器（改了代码后最常用） |
| **`d`** | **删除/移除** (Remove) | 移除容器 |
| **`e`** | **进入 Shell** (Exec) | 相当于 `docker exec -it <id> /bin/sh` |
| **`b`** | **批量操作** | 打开批量菜单（如重启所有容器） |

#### 3. 🖼️ 镜像操作 (Images 面板)
| 快捷键 | 作用 | 说明 |
| :--- | :--- | :--- |
| **`d`** | **删除镜像** | 清理不用的镜像 |
| **`p`** | **拉取** (Pull) | 更新选中的镜像 |

#### 4. 🧹 清理与维护
| 快捷键 | 作用 | 说明 |
| :--- | :--- | :--- |
| **`D`** (大写) | **清理系统** (Prune) | 删除所有停止的容器、未使用的网络等 |

---

### 💡 给你的建议 (Best Practice)

1.  **忘记了就按 `?`**：这两个工具的帮助菜单做得非常好，按 `?` 可以看到当前面板能做的所有操作。
2.  **筛选模式**：在很长的列表（比如几十个分支或容器）里，按下 **`/`** 可以输入关键词进行搜索过滤。
3.  **鼠标也能用**：虽然我们追求全键盘，但 LazyGit 和 LazyDocker 实际上是支持鼠标点击的，刚开始不熟练可以用鼠标点面板。

你现在的截图里，**Files** 面板处于选中状态，你只需要按 **`Space`** 选中文件，然后按 **`c`** 就可以开始写提交信息了，非常流畅！
